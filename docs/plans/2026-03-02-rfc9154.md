# RFC 9154 Compliance Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make the Nominet EPP module comply with RFC 9154 (Secure Authorization Information for Transfer) ahead of Nominet's June production deadline.

**Architecture:** Two targeted changes — declare the RFC 9154 namespace in the EPP login command, and replace the weak `mt_rand`-based `generatePassword()` with a CSPRNG implementation producing 25-character alphanumeric codes (~148 bits entropy).

**Tech Stack:** PHP, metaregistrar/php-epp-client, Blesta module framework

**Design doc:** `docs/plans/2026-03-02-rfc9154-design.md`

### Task 2: Replace `generatePassword()` with CSPRNG implementation

**Files:**
- Modify: `nominet.php:1022-1034`

**Context:**
`generatePassword()` is called in two places:
- `createContact()` (~line 1996): sets the EPP contact auth code
- `registerDomain()` (~line 2018): sets the domain auth code at registration

Current implementation uses `mt_rand` (not cryptographically secure) and produces 6–14 char codes (~47 bits max entropy). RFC 9154 requires ≥128 bits. The fix uses `random_bytes()` with rejection sampling to eliminate modular bias, producing 25-char alphanumeric codes (~148 bits).

The method signature `generatePassword($min_length, $max_length)` is preserved so call sites need no changes. The length parameters are silently ignored — RFC 9154 compliance overrides them.

**Step 1: Replace the method body**

Replace the entire `generatePassword()` method body (keeping the signature):

```php
private function generatePassword($min_length = 10, $max_length = 14)
{
    // RFC 9154: auth codes must have >=128 bits of entropy.
    // 25 alphanumeric chars = log2(62^25) ≈ 148 bits.
    // Rejection sampling (accept bytes 0-247 only) eliminates modular bias.
    $pool = 'abcdefghijklmnopqrstuvwxyzABCDEFGHIJKLMNOPQRSTUVWXYZ0123456789';
    $result = '';

    while (strlen($result) < 25) {
        $byte = ord(random_bytes(1));
        if ($byte < 248) {
            $result .= $pool[$byte % 62];
        }
    }

    return $result;
}
```
